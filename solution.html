Javascript Solution

true + false

Selon la spécification du langage ECMAScript, les deux valeurs booléennes sont converties en leurs homologues numériques

01/25

[,,,].length

génère un tableau avec trois emplacements vides. La dernière virgule est une virgule de fin.

02/25

[1, 2, 3] + [4, 5, 6]

The extremely simplified answer is that the arrays are converted to strings and are then concatenated. See Denys Dovhan's explanation for how this happens. To learn more about this behavior, visit this StackOverflow answer for a mid-level explanation or this blog post for a more detailed one.

03/25

0.2 + 0.1 === 0.3

Il s'agit d'un dilemme standard de comparaison de valeurs à virgule flottante. Au lieu de comparer directement deux virgules flottantes, il faut comparer les virgules flottantes avec un certain niveau de tolérance.

04/25

10,2

L'opérateur virgule renvoie simplement la valeur du dernier opérande.

05/25

!!""

Vous pouvez ajouter deux points d'exclamation avant toute valeur pour obtenir sa primitive booléenne correspondante. Le terme technique pour cela est double NOT (oui, vraiment, Borat adorerait ça). La conversion est basée sur la véracité ou la fausseté de la valeur.

06/25

+!![]

Les tableaux sont véridiques, donc le double opérateur NOT affichera vrai. Le caractère plus convertit alors vrai en sa représentation numérique : 1.

07/25

parseInt (0.0000005)

La fonction parseInt convertit son premier argument en une chaîne (si ce n'est pas déjà le cas), puis en un nombre. Lorsque 0.0000005 est converti en chaîne

08/25

true == "true"

Selon les règles de comparaison d'égalité abstraite, ces deux valeurs sont converties en nombres lors de la comparaison.

09/25

010 - 03

010 est traité comme un nombre octal par JavaScript. Ainsi, sa valeur est en base 8. Le MDN Web Docs fournit une brève explication des nombres octaux.

10/25

"" - - ""

Ces deux chaînes vides sont toutes deux converties en 0.

11/25

null + 0

Null convertit en sa représentation numérique

12/25

0/0

Comme il n'y a pas de réponse numérique significative à l'équation 0/0, la sortie est simplement NaN.

13/25

1/0 > 10 ** 1000

JavaScript traite ces deux valeurs comme infinies, et l'infini est égal à l'infini. En savoir plus sur les infinis.

14/25

true++

Notre première et unique erreur de syntaxe. J'ai proposé l'option SyntaxError pour un grand nombre de questions, en espérant que certains utilisateurs trouveraient une syntaxe si bizarre qu'elle ne pourrait être autorisée. J'ai donc estimé que je devais ajouter au moins une expression qui entraîne effectivement une SyntaxError.

15/25

"" - 1

Alors que l'opérateur d'addition (+) est utilisé à la fois pour les nombres et les chaînes, l'opérateur de soustraction (-) n'a aucune utilité pour les chaînes, donc JavaScript interprète cela comme une opération entre des nombres. Une chaîne vide est de type contraint à 0.

16/25

(null - 0) + "0"

null est contraint à 0

17/25

true + ("true" - 0)

On pourrait penser que le langage JS est tellement banal qu'il convertit la chaîne de caractères "true" en représentation numérique du booléen true. Mais ce n'est pas tout à fait le cas. Ce qui se passe en fait, c'est qu'il essaie de convertir la chaîne de caractères en un nombre et échoue.

18/25

!5 + !5

Placer un point d'exclamation solennel, également connu sous le nom d'opérateur logique NOT, avant une valeur non booléenne la convertira en une valeur booléenne opposée à ce que la fonction booléenne la convertirait.

19/25

[] + []

Encore une fois, la réponse extrêmement simplifiée est que JavaScript convertit les tableaux en chaînes. Faites défiler jusqu'à la question 3 pour trouver des ressources qui expliquent ce comportement.

20/25

1 + 2 + "3"

JavaScript exécutera ces opérations de gauche à droite. La concaténation de chaînes aura la priorité lorsque le nombre trois est ajouté à la chaîne trois

21/25

typeof NaN

La spécification du langage ECMAScript explique NaN comme une valeur numérique qui est une valeur IEEE 754 "Not-a-Number". Cela peut sembler étrange, mais c'est un principe commun en informatique

22/25

undefined + false

Alors que false peut être converti en nombre, undefined ne le peut pas.

23/25

"" && -0

Il est fort probable que vous n'ayez jamais utilisé l'opérateur logique AND que dans des expressions if, mais lorsqu'il est utilisé seul, il renvoie en fait la valeur de l'un des opérandes. Si la première expression est vraie, il renvoie la seconde. Sinon, il renvoie la première. Une chaîne vide est fausse, elle renvoie donc le premier opérande.

24/25

+!!NaN * "" - - [,]

La finale combine certaines des syntaxes bizarres que j'ai abordées dans ce quiz. J'ai déjà expliqué tout ce comportement dans les explications ci-dessus, à l'exception de l'opérateur de multiplication (*), qui transformera la chaîne vide en son équivalent numérique : 0..
